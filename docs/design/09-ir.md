# 9. Intermediate Representation (IR)

MLang uses a single mid-level IR called **MIR**: a typed, control-flow-graph,
SSA-form representation. MIR sits between the typed AST and LLVM IR. It is where
all language-aware optimization happens.

## 9.1 Why a custom IR (and not "just lower to LLVM")

LLVM IR is an excellent *target-level* IR, but it has already thrown away the
facts MLang cares about: nullability, ownership, which calls are ARC operations,
which `match`es were exhaustive, generic shapes, and bounds-check provenance.
Optimizing on LLVM IR would mean re-deriving those facts (expensively and
incompletely). MIR keeps them as first-class, so passes like ARC elision, bounds-
check elimination, and devirtualization are precise and cheap. Once MIR is fully
optimized, lowering to LLVM is mechanical.

## 9.2 Structure

A MIR `Function` is a list of basic `Block`s. Each block is a list of
`Instruction`s ending in a single `Terminator`. Values are in SSA form: each is
defined exactly once; control-flow merges use `phi` nodes.

```
fn @area(%shape: *Shape) -> f64 {
  bb0:
    %tag = load_tag %shape
    switch %tag [ Circle: bb1, Rectangle: bb2 ]
  bb1:
    %c   = downcast %shape as *Circle
    %r   = field %c, .radius            ; f64
    %r2  = fmul %r, %r
    %pi  = const 3.14159 : f64
    %a1  = fmul %pi, %r2
    br bb3(%a1)
  bb2:
    %rc  = downcast %shape as *Rectangle
    %w   = field %rc, .width
    %h   = field %rc, .height
    %a2  = fmul %w, %h
    br bb3(%a2)
  bb3(%result: f64):
    ret %result
}
```

Key instruction families:

- **Pure**: arithmetic, comparison, `field`, `index`, `cast`, `const`.
- **Memory**: `load`, `store`, `alloca` (stack slot), `gep`-like `field`/`index`.
- **Ownership**: `retain`, `release`, `drop`, `move`, `borrow`, `end_borrow`.
  These are explicit in MIR (inserted in Chapter 8) so the optimizer can reason
  about and remove them.
- **Calls**: `call` (static), `call_virtual` (interface/virtual dispatch),
  `call_async` (coroutine resume points), `call_intrinsic`.
- **Terminators**: `br`, `cond_br`, `switch`, `ret`, `unreachable`, `unwind`
  (for exception edges, Chapter 17).

Every value carries its MIR type, which is a lowered form of the semantic type:
generics are already monomorphized, nullable reference types are pointers with a
nullability bit, value types are by-value aggregates.

## 9.3 SSA construction

MIR is built directly in SSA using the standard "SSA construction on the fly"
algorithm (Braun et al.): local variables become SSA values, and `phi`s are
inserted lazily at control-flow joins as variables are read. This avoids building
a non-SSA CFG and then running a separate dominance-frontier SSA pass; SSA is the
only form MIR ever has.

## 9.4 Type and effect annotations

MIR values keep flags the optimizer consumes:

- `nonnull` on pointer values whose nullability analysis proved present.
- `noalias` on values from unique-ownership or exclusive borrows.
- `unique` (refcount is provably 1) which unlocks in-place mutation of COW data.
- `pure`/`readonly`/`readnone` on calls, propagated from the callee's summary.

These map almost directly onto LLVM attributes at codegen time, so the work the
MLang optimizer does is preserved for LLVM rather than re-discovered.

## 9.5 Textual and binary forms

MIR has a textual form (shown above) for `--emit=ir` and for writing optimizer
unit tests as "input MIR -> expected MIR" golden files. It also has a compact
binary form (`.mir`) used for the on-disk cache and for cross-function inlining
across modules (the inliner can pull a callee's MIR from its `.mmi`).

## 9.6 Verifier

A MIR **verifier** runs (in debug builds and in CI) after construction and after
every pass. It checks SSA dominance (every use is dominated by its def), type
consistency of every instruction, well-formed terminators and `phi`s, and
ownership balance (every `retain` has a matching `release` on every path, every
owned value is dropped or moved exactly once). A pass that produces invalid MIR
fails the verifier immediately, which is how optimizer bugs are caught at their
source instead of as miscompiles downstream.
